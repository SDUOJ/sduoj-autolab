## 1. 功能概述

### 1.1 核心功能

本系统实现了以课程为核心的完整实验课程管理、考勤与座位管理功能，主要包括：

**课程管理模块**：
- ✅ 课程创建（绑定用户组、设置课程信息）
- ✅ 课程时间管理（多次课程、课程内容、资料、作业）
- ✅ 课程教室分配（自动分配座位）
- ✅ 课程考勤自动创建

**座位管理模块**：
- ✅ 教室管理（创建、编辑、查询）
- ✅ 座位状态管理（禁用标记、备注原因）
- ✅ 学生座位分配（基于课程）
- ✅ 助教管理（分配助教、联系方式、备注）

**考勤管理模块**：
- ✅ 手动考勤记录（支持多种状态）
- ✅ 二维码考勤（快速签到）
- ✅ 请假管理（文件上传、审批流程）
- ✅ 考勤统计（多维度查询、状态筛选）

### 1.2 技术栈

- **后端框架**: FastAPI (Python)
- **数据库**: MySQL (SQLAlchemy ORM)
- **文件处理**: SDUOJ文件接口
- **认证**: Token based authentication

### 1.3 核心特性

- **以课程为中心**: 所有功能围绕课程展开，统一管理
- **灵活的考勤方式**: 支持手动记录和二维码两种方式
- **完善的请假流程**: 支持文件上传和审批
- **自动化管理**: 自动分配座位、自动创建考勤

## 3. 数据库设计

### 3.1 教室管理相关表

#### 3.1.1 教室表 (oj_class)

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| c_id | INTEGER | 教室ID | 主键，自增 |
| c_name | VARCHAR(50) | 教室名称 | 唯一，索引 |
| address | VARCHAR(50) | 教室地点 | 非空 |
| c_seat_num | INTEGER | 座位数量 | 非空 |
| ext_config | JSON | 扩展配置 | 可空 |

**用途**: 存储所有教室的基本信息，支持多教室管理

**ext_config 结构**:
```json
{
  "seat-ban": [
    {"seat_number": 5, "reason": "座位损坏"},
    {"seat_number": 12, "reason": "电脑故障"}
  ]
}
```

### 3.2 课程管理相关表

#### 3.2.1 课程表 (oj_course)

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| course_id | BIGINT | 课程ID | 主键，自增 |
| course_name | VARCHAR(100) | 课程名称 | 非空 |
| group_id | BIGINT | 用户组ID | 非空，索引 |
| tag | VARCHAR(20) | 课程标签 | 非空 |
| c_ids | JSON | 教室ID列表 | 可空，存储教室ID数组 |
| create_time | DATETIME | 创建时间 | 默认当前时间 |
| ext_config | JSON | 扩展配置 | 可空 |

**用途**: 存储实验课程基本信息，作为整个系统的核心关联表

**tag 可选值**: `授课`、`实验`、`考试`、`答疑`等

**c_ids 结构**:
```json
[1, 2, 3]
```
- 存储教室ID数组，支持课程使用多个教室
- 例如：实验课可能需要在多个实验室分批进行

**ext_config 结构**:
```json
{
  "description": "Java程序设计实验课",
  "semester": "2024春季学期",
  "auto_assign_seat": true,
  "auto_create_checkin": true
}
```

#### 3.2.2 课程时间表 (oj_course_schedule)

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| schedule_id | BIGINT | 课程时间ID | 主键，自增 |
| course_id | BIGINT | 课程ID | 外键 → oj_course.course_id |
| sequence | INTEGER | 课程序号 | 非空 |
| start_time | DATETIME | 课程开始时间 | 非空 |
| end_time | DATETIME | 课程结束时间 | 非空 |
| course_content | TEXT | 课程内容 | 可空 |
| course_materials | JSON | 课程资料 | 可空 |
| course_homework | TEXT | 课程作业 | 可空 |
| sg_id | INTEGER | 签到ID | 外键 → oj_sign.sg_id，可空 |

**联合唯一约束**: (course_id, sequence)

**用途**: 记录课程的每次上课安排和相关信息

**course_materials 结构**:
```json
{
  "slides": ["file_id_1", "file_id_2"],
  "codes": ["file_id_3"],
  "references": ["file_id_4"]
}
```
- 所有文件ID使用SDUOJ文件接口

### 3.3 考勤管理相关表

#### 3.3.1 考勤表 (oj_sign)

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| sg_id | INTEGER | 考勤ID | 主键，自增 |
| course_id | BIGINT | 课程ID | 外键 → oj_course.course_id |
| schedule_id | BIGINT | 课程时间ID | 外键 → oj_course_schedule.schedule_id，唯一 |
| title | VARCHAR(63) | 考勤标题 | 非空 |
| mode | INTEGER | 考勤模式 | 非空 |
| u_gmt_create | DATETIME | 创建时间 | 默认当前时间 |
| u_gmt_modified | DATETIME | 修改时间 | 非空 |
| sign_is_deleted | INTEGER | 删除标记 | 0=正常, 1=已删除 |

**用途**: 存储考勤活动的配置信息，绑定到具体的课程时间

**说明**: 
- 考勤记录不能单独创建，必须绑定到 oj_course_schedule 表的某一节课
- schedule_id 为唯一约束，一节课只能有一个考勤记录
- 考勤时间范围由 course_schedule 的 start_time 和 end_time 决定

**mode 说明**:
- `0`: 手动考勤
- `1`: 二维码考勤

#### 3.3.2 用户考勤表 (oj_sign_user)

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| sg_u_id | INTEGER | 学生考勤ID | 主键，自增 |
| username | VARCHAR(63) | 学生用户名 | 非空 |
| sg_id | INTEGER | 考勤ID | 外键 → oj_sign.sg_id |
| status | INTEGER | 考勤状态 | 非空，默认0 |
| sg_time | DATETIME | 签到时间 | 可空 |
| seat_number | TINYINT | 座位号 | 可空 |
| token | VARCHAR(63) | 签到凭证 | 唯一 |
| leave_files | JSON | 请假文件列表 | 可空 |
| leave_message | TEXT | 请假说明 | 可空 |
| leave_status | INTEGER | 请假审批状态 | 可空 |
| is_deleted | INTEGER | 删除标记 | 可空 |

**联合唯一约束**: (sg_id, username)

**用途**: 记录每个学生的考勤详细信息

**status 状态说明**:
- `0`: 无记录（默认）
- `1`: 出勤
- `2`: 缺勤
- `3`: 迟到/早退
- `4`: 请假（已批准）
- `5`: 请假申请中

**leave_files 结构**:
```json
["file_id_1", "file_id_2", "file_id_3"]
```
- 所有文件ID使用SDUOJ文件接口

**leave_status 说明**:
- `NULL`: 未提交请假
- `0`: 请假申请中
- `1`: 请假批准
- `2`: 请假拒绝

**重新申请逻辑**:
- 当 leave_status 为 `NULL` 或 `2`（拒绝）时，学生可以重新提交请假申请
- 重新申请会覆盖之前的 leave_files 和 leave_message
- 重新申请后 leave_status 重置为 `0`（申请中）

## 4. 课程管理功能

### 4.1 创建课程

**业务流程**:
```
1. 创建课程基本信息（绑定用户组）
2. 设置课程标签（授课/实验/考试等）
3. 可选：分配教室
4. 可选：开启自动分配座位和自动创建考勤
```

**请求参数**:
```python
{
    "course_name": "Java程序设计实验",
    "group_id": 100,
    "tag": "实验",
    "c_ids": [1, 2],  # 可选，支持多个教室
    "ext_config": {
        "description": "Java程序设计实验课",
        "semester": "2024春季学期",
        "auto_assign_seat": true,
        "auto_create_checkin": true
    }
}
```

### 4.2 课程时间管理

#### 4.2.1 添加课程时间

**业务流程**:
```
1. 指定课程ID和课程序号
2. 设置上课时间
3. 填写课程内容和作业
4. 上传课程资料（文件ID列表）
5. 可选：自动创建对应的考勤记录
```

**请求参数**:
```python
{
    "course_id": 1,
    "sequence": 1,
    "start_time": "2024-03-01 14:00:00",
    "end_time": "2024-03-01 16:00:00",
    "course_content": "第一章：Java基础",
    "course_materials": {
        "slides": ["file_id_1", "file_id_2"],
        "codes": ["file_id_3"]
    },
    "course_homework": "完成课后练习1-5题",
    "auto_create_checkin": true  # 可选：自动创建考勤记录
}
```

#### 4.2.2 编辑课程时间

**支持的操作**:
- 修改上课时间（start_time, end_time）
- 更新课程内容
- 添加/删除课程资料
- 修改课程作业
- 创建考勤记录（如尚未创建）

### 4.3 课程教室与座位管理

#### 4.3.1 分配教室

**业务流程**:
```
1. 为课程指定一个或多个教室
2. 系统检查所有教室可用性
3. 如开启自动分配座位，则：
   - 获取课程用户组成员
   - 按教室分批自动分配座位号
   - 避开禁用座位
   - 优先填满第一个教室，再使用后续教室
```

**请求参数**:
```python
{
    "course_id": 1,
    "c_ids": [1, 2, 3]  # 可以分配多个教室
}
```

#### 4.3.2 手动分配座位

**请求参数**:
```python
{
    "course_id": 1,
    "username": "2020001",
    "c_id": 1,
    "seat_number": 10
}
```

### 4.4 助教管理

#### 4.4.1 添加助教

## 6. 考勤管理功能

### 6.1 考勤方式

#### 6.1.1 方式一：手动考勤记录

**特点**:
- 教师/助教手动记录学生出勤情况
- 支持批量记录
- 每次可以只记录部分学生
- 支持多种考勤状态

**请求参数**:
```python
{
    "sg_id": 1,
    "records": [
        {"username": "2020001", "status": 1, "seat_number": 10},
        {"username": "2020002", "status": 2},
        {"username": "2020003", "status": 3}
    ]
}
```

**状态说明**:
- `0`: 无记录（默认，不修改）
- `1`: 出勤
- `2`: 缺勤
- `3`: 迟到/早退
- `4`: 请假（已批准）
- `5`: 请假申请中

#### 6.1.2 方式二：二维码考勤

**步骤1: 生成Token**

API: `POST /checkin/generateToken`

```python
{
    "username": "2020001",
    "sg_id": 1
}
```

**返回**:
```json
{
    "token": "a1b2c3d4e5f6..."
}
```

**步骤2: 扫描验证**

API: `POST /checkin/verifyToken`

```python
{
    "token": "a1b2c3d4e5f6...",
    "seat_number": 10
}
```

**业务逻辑**:
```
1. 学生端生成唯一Token
2. 显示二维码
3. 教师端扫描验证
4. 自动标记出勤状态
```

### 6.2 请假管理

#### 6.2.1 提交请假

**API**: `POST /checkin/submitLeave`

**请求参数**:
```python
{
    "sg_u_id": 123,
    "leave_files": ["file_id_1", "file_id_2"],  # SDUOJ文件ID列表
    "leave_message": "因病请假，已上传医院诊断证明和病假条"
}
```

**业务逻辑**:
```
1. 检查 leave_status：必须为 NULL 或 2（拒绝）才能申请
2. 提交请假申请，上传相关证明文件
3. 如果是重新申请，覆盖之前的 leave_files 和 leave_message
4. status 自动设置为 5（请假申请中）
5. leave_status 设置为 0（申请中）
```

#### 6.2.2 审批请假

**API**: `POST /checkin/approveLeave`

**请求参数**:
```python
{
    "sg_u_id": 123,
    "leave_status": 1,  # 1=批准, 2=拒绝
    "remark": "已批准，注意休息"
}
```

**业务逻辑**:
```
1. 教师/助教审批请假
2. 如批准：status设置为4（请假已批准）
3. 如拒绝：status保持原状态或设置为2（缺勤）
4. 更新leave_status
```

### 6.3 考勤查询与统计

#### 6.3.1 按课程查询

**API**: `GET /checkin/course/{course_id}/statistics`

**返回数据**:
```json

    "course_id": 1,

    "total_students": 50,

        "attended": 45,

        {
            "sequence": 1,

            "end_time": "2024-03-01 16:00:00",

            "attended": 45
        }

```

#### 6.3.2 按学生查询

**API**: `GET /checkin/student/{username}/records`

**参数**:

## 7. 业务流程

### 7.1 完整课程流程

```
┌─────────────────────────────────────────────────────┐
│              课程准备阶段（教师端）                    │
└─────────────────────────────────────────────────────┘
                         ↓
  ┌──────────────┬───────┴───────┬──────────────┐
  │              │               │              │
  ▼              ▼               ▼              ▼
创建课程      添加课程时间    分配教室       添加助教
  │              │               │              │
  └──────────────┴───────┬───────┴──────────────┘
                         ↓
              系统自动分配座位（可选）
                         ↓
              系统自动创建考勤（可选）
                         │
┌────────────────────────┴────────────────────────────┐
│              上课当天（学生端）                        │
└─────────────────────────────────────────────────────┘
                         ↓
           ┌─────────────┼─────────────┐
           ▼             ▼             ▼
      二维码考勤    手动考勤记录   提交请假
           │             │             │
           └─────────────┼─────────────┘
                         │
┌────────────────────────┴────────────────────────────┐
│              课后管理（教师端）                        │
└─────────────────────────────────────────────────────┘
                         ↓
           ┌─────────────┼─────────────┐
           ▼             ▼             ▼
      审批请假      查询统计      导出报表
```

### 7.2 考勤记录流程

```
教师端                          服务器                      学生端
  │                              │                           │
  │ 1. 为课程时间创建考勤            │                           │
  ├──────────────────────────────>│                           │
  │  (绑定到 schedule_id)         │ 初始化所有学生记录           │
  │                              │ status=0 (无记录)          │
  │                              │                           │
  │ 2a. 手动记录考勤（部分学生）     │                           │
  ├──────────────────────────────>│                           │
  │  [{username, status}]        │ 更新指定学生状态            │
  │                              │                           │
  │                              │      2b. 二维码签到        │
  │                              │<──────────────────────────┤
  │                              │      生成Token             │
  │                              │      返回二维码            │
  │                              ├──────────────────────────>│
  │ 3. 扫描验证                    │                           │
  │<──────────────────────────────┤                           │
  │  验证成功，更新状态=1           │                           │
  │                              │                           │
  │ 4. 查询考勤统计                 │                           │
  ├──────────────────────────────>│                           │
  │                              │ 返回各状态人数              │
  │<──────────────────────────────┤                           │
```
